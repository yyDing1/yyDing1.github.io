Codeforces1368B

难度：1500

题目大意

编一个字符串，使得$codeforces$单词在这个字符串的子列中至少出现了$k$次

解题思路

暴力+贪心，采取”$ccc…ooo…ddd…eee…fff…ooo…rrr…ccc…eee…sss$”这样的编码形式肯定是最优的，假设各个字母出现次数是$a_0, a_1, …, a_9$次，那么$all = \prod_{i = 0}^9{a_i}$。$a_{0 - 9}$越接近时$all$值最大。所以枚举得到满足$p^{10} >= k$最小的$p$，再进行放缩，使其值刚好大于等于$k$。

(第一眼以为是正好出现$k$次，质因数分解了半天$qwq$)

Codeforces1326D2

难度：1800

题目大意

给定一个字符串$s$，求取其一个前缀$t_1$一个后缀$t_2$能组成的最长回文串$t_1 + t_2$。(前缀或后缀可以为空，$t_1 + t_2$的长度必须小于等于原字符串的长度)。

解题思路

可以很快推断出

(这里的$+$是字符串的拼接)

最大回文串一定是由最长回文前后缀和一个回文串($w$)组成。(存在其他情况，但是可以归结为此种情况，如$abccbaba$，你可以理解为只有一个前缀，也可理解为是最长回文前后缀加一个回文串)。

首先匹配完最长回文前后缀。$s_i = s_{n - 1 - i}$

之后去掉一些字符，找剩下的字符串$m$前(后)缀中最长回文串，注意这里的$w$只能是剩下字符串的一个前缀或者后缀。

这里有3种2种解决方法：

  1. $manacher$算法。经典的$O(n)$回文子串算法。$qwq$本菜鸡还不会
  2. $hash$ + 二分。存取$O(n)$，查找$O(logn)$。个人感觉可以，没有试验过。不知道会不会被卡。
  3. 用$kmp$中求$nxt$数组的部分，因为要求的回文子串的特殊性，贴合$nxt$数组维护的是当前最长公共前后缀，故
    • 最长回文前缀 $= getnxt(m + / + \overline{m})$
    • 最长回文后缀 $= getnxt(\overline{m} + / + m)$
    • $\overline{m}$指$m$的逆置，中间加的’/‘只要是$m$和$\overline{m}$中没有出现过的符号就行。
    • 两者相对比就可以得到中间的最长回文子串$w$，感觉是很新颖且巧妙的方法。

Codeforces1290B

难度：1800

题目大意

对于给定的一个长串$s$，每次访问访问其中的一个连续子串，问它是否存在一个$irreducible \quad anagram$。具体的定义感觉有点难解释，看原题吧。

解题思路

从数据范围可以看出，每次访问必须是$O(1)$的。

可以归得出一下结论(以下结论是递进关系，即后一个在前面都不成立的前提下)：

  1. 如果仅访问一个字母(即$l = r$)，根据定义他自己就是(存在)。
  2. 如果$s_l \neq s_r$，那么很显然，只需将$s_l$和$s_r$交换一下其他字母不动就可以组成一个$irreducible$ $anagram$
  3. 如果出现了区间内出现了三种及三种以上的字母(如$a…b…c…a$)，只需间隔互换一次即可($a…c…b…a$)，其他字母不动，就可以组成一个$irreducible$ $anagram$。
  4. 剩下的情况：只出现两种字母且$s_l = s_r$(如$abbabbbabba$)。证明这个情况都是不存在$irreducible$ $anagram$的。这个证明稍有些复杂，下面是我的简易证明。方便起见两种不同字母用$a$，$b$替代($a$是两端的字母，$n$是访问的串的长度)。
    • 很显然，若$t$的两端还出现$a$，那必然不是$irreducible$ $anagram$
    • 设序偶$f_{string}(i) = (x, y)$表示长度为$i$时$a$出现了$x$次，$b$出现了$y$次($x + y = i$)。设访问串内共有$n$个$a$，$m$个$b$。
    • 两端都是$b$
      • $f_s(1) = (1, 0)$，$f_s(n - 1) = (n - 1, m)$
      • $f_t(1) = (0, 1)$，$f_t(n - 1) = (n, m - 1)$
    • 可以将其映射到二维直角坐标系下，$(1, 0)$和$(0, 1)$是两个起点，$(n - 1, m)$和$(n, m - 1)$是对应的两个终点，每次向上或向由走，两条路线肯定有交点。
    • 所以必定存在一种划分，使得$s$和$t$的每一个部分都互为$anagrams$。
    • 所以剩下的情况都不存在$irreducible$ $anagram$

至于多次询问，算出每个字母的前缀和。每个询问做一次差就可知道具体情况。每次询问复杂度都是常数级。

Codeforces1256F

难度：2000

题目大意

给定两个长度相同字符串$s$和$t$，每次操作可以对两个字符串中相同长度的子串(位置可以不同)进行逆转。问两个字符串是否可以通过很多次这样的操作变成相同的字符串。

解题思路

可以得出以下结论(以下结论是递进关系，即后一个在前面都不成立的前提下)：

核心思想：任何一个串的逆转可以拆解成很多次一个个长度为2的串的逆转。所以只要考虑相邻字符间的交换即可。

  1. 很显然，如果两个字符串含有的相同字母数量不相等，那一定不可能。
  2. 如果字符串含有相同字母，那么一定可以，下面是简单证明。
    • 不断对$s$进行同一个子串的长度为2的逆转操作(相当于交换两个相邻的字符)，显然偶数次逆转后还是原串$s$。
    • 对$t$进行相邻字符的交换操作，直至其变为原本的$s$。
    • 若此时交换次数为偶数，则两者相等，成立。
    • 若此时交换次数为奇数，则选取$t$中两个相同的字符，通过一系列两两交换这操作使得这两个字符交换。
    • 使得$t_i$，$t_j$交换位置，需要重复交换相邻字符$2\times (i - j) - 1$次，对$t$的总操作次数为奇数+奇数=偶数。成立。
  3. 如果字符串中不含相同字母，这个字符串长度最大为26，这就大大减小了数据范围的约束。从两个串的逆序数入手。得出逆序数相差偶数可以，逆序数为奇数不可以。下面是证明。
    • 不断对$s$进行同一个子串的长度为2的逆转操作(相当于交换两个相邻的字符)，显然偶数次逆转后还是原串$s$。
    • 每次交换相邻字符，会使得整个串的逆序数加1或减1。
    • $t$是肯定可以通过有限次相邻字符的逆转变成原串$s$的。
    • 若这个逆转次数为偶数，那么不断做相同变换的$s$会是最原先的$s$串。
    • 若这个逆转次数为奇数，那么$t$最终会呈现出状态会刚好和$s$的状态错开(如$s: abcd$，$t: abdc$)，肯定无法变成相同的串。

Nowcoder????A

难度：好难($qwq$)

原题翻译

给定$n$个字符串$s_1, s_2, s_3, …, s_n$。定义$f(s, t)$为满足$s_{1…t} = s_{|t| - i + 1…t}$的最大值$i$，如果这样的$i$不存在，那么$f(s, t) = 0$，现在要算以下算式的值。

解题思路(official)

$hash + kmp$

  1. 首先，前缀的总数量和后缀的总数量都是$O(\sum|s_i|)$的，所以可以先把每个后缀的$hash$值存起来。
  2. 然后枚举每个串$s$的每个前缀$s_{1..i}$，有多少个后缀和枚举的这个前缀相同，记为$cnt[i]$，则可以说明有$cnt[i]$个串的后$i$位与$s$的前$i$位相同。
  3. 但这个不能直接拿来算答案，因为一个串可能会给Cnt[]数组带来大于1的贡献。
  4. 现在考虑去重，比如两个$aba$，其中会有两个前缀$a$，$aba$被算到，但实际上只应该取$aba$而不是$a$。所以我们考虑把每个串的$nxt$数组求出来，求完$nxt$数组之后再从前往后$cnt[nxt[i]] -= cnt[i]$即完成去重，然后直接算答案即可。
  5. 时间复杂度: $O(\sum|s_i|)$
