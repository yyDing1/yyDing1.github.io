Trie Tree

Posted on 2020-07-04 |

字典树，无内鬼。

简介

Trie Tree：字典树，前缀树

trie是一个以空间换时间的算法

网络上关于字典树的描述不一，可以将边作为字符，也可将节点作为字符。(感觉把节点作为字符图好画点)。

  • 字典树的基础定义部分很简单，如下是一颗包含单词ARE, AS, DOT1, NEWS, NOT, ZEN的字典树(网图，‘1’乱入了)。
  • 字典树通过tree[当前节点][当前字符]来访问下一个节点，在插入时已经对于字典树的节点进行好了编号。(具体怎么表示看下面的代码部分)。
  • 每一个字符一个节点，字典树的总结点数小于等于总字母数。可作为对tree进行初始化时的参考数值。

为什么用字典树

  • 存取：有公用节点，一定程度上节省了存储空间
  • 查找：在n个平均长度为k的字符串中去查找给定的字符串是否存在。
    • 对于单次查找
      • 顺序查找：读取O(nk)，查找O(nk)。总O(nk)。
      • 字典树查找：读取O(nk)，查找O(k)。总O(nk)。
    • 对于m次查找
      • 顺序查找：读取O(nk)，查找O(nkm)。总O(nkm)。
      • 字典树查找：读取O(nk)，查找O(mk)。总O(max(n, m)k)。

当n，m较大时，字典树的查找具有很大优势。当然字典树也是AC自动机实现的前提。

当然对前缀进行查找时字典树的更显优势，详见下文的题hdu1251。

具体操作

具体来说，Trie一般支持两个操作：

  1. Trie.insert(W)：第一个操作是插入操作，就是将一个字符串W加入到集合中。

  1. Trie.search(S)：第二个操作是查询操作，就是查询一个字符串S是不是在集合中。

数组含义：

  1. int tree[MAX_NODE][CHARSET]：树状数组(MAX_NODE：字母总数, CHARSET：字符集的字母数)
  2. int color[MAX_NODE]：节点被染的颜色(color[i] == 1：存在以i节点为末尾的单词, 反之不存在)
  3. int cnt[MAX_NODE]：节点被访问数(前缀数)

代码优化：

多组数据时，如果每次都进行以下操作，将所有节点数组归0，显然有很多是无用操作，t较大时很有可能会导致tle，所以可以在使用过程中进行memset。

思路：不断去申请新的节点，每次申请将该节点连接的字符都归0，避免了做多余操作。(就不用了一开始memset整个tree数组了)。

需要注意一开始需要初始化0行
