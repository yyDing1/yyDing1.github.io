Trie Tree

Posted on 2020-07-04

简介

具体来说，Trie一般支持两个操作：

  1. Trie.insert(W)：第一个操作是插入操作，就是将一个字符串W加入到集合中。
  2. Trie.search(S)：第二个操作是查询操作，就是查询一个字符串S是不是在集合中。

trie是一个以空间换时间的算法

网络上关于字典树的描述不一，可以将边作为字符，也可将节点作为字符。(主要方便理解)

含义：

  1. int tree[MAX_NODE][CHARSET]：树状数组(MAX_NODE：字母总数, CHARSET：字符集的字母数)
  2. int color[MAX_NODE]：节点被染的颜色(color[i] == 1：存在以i节点为末尾的单词, 反之不存在)
  3. int cnt[MAX_NODE]：节点被访问数(前缀数)

Attention！Attention！Attention！

用C风格的字符串！用C风格的字符串！用C风格的字符串！

不要用string！不要用string！不要用string！

亲身感受：因为用了string，t了一个下午找不出问题(t到怀疑人生)，直到有大佬指出string + cin容易t。

代码优化：

多组数据时，如果每次都进行以下操作，将所有节点数组归0，显然有很多是无用操作，t较大时很有可能会导致tle，所以可以在使用过程中进行memset。

思路：不断去申请新的节点，每次申请将该节点连接的字符都归0，避免了做多余操作。
