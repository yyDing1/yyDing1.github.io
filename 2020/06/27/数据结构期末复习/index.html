Posted on 2020-06-27

第一部分 绪论

  • 数据，数据元素，数据项，数据对象的定义。

  • Data_Structures = (D, R)。其中: D是数据元素的有限集合，R是D上关系的有限集合。结构 = 实体 + 关系。

  • 程序 = 算法 + 数据结构

  • 数据结构通常包括2个层次：

    1. 逻辑结构：元素之间的关系，与计算机无关
    2. 存储结构：数据结构的存储，与计算机相关 $数据结构=\left{\begin{matrix} 逻辑结构=\left{\begin{matrix}线性结构(List)\ 树形结构(Tree)\ 图状(Graph)\ 集合(set)\end{matrix}\right.\ \ 存储结构=\left{\begin{matrix}顺序存储\ 链式存储\ 索引存储\ 哈希(散列)存储\end{matrix}\right.\end{matrix}\right.$

  • Python内置数据结构中:

    1. 线性数据结构：list, turple, str
    2. 集合数据结构：set, dirt

  • ADT：Abstract Data Type(抽象数据类型)

  • ABC：Abstract Base Class(抽象基类)

  • 子类与父类方法的继承与覆盖：
    如果子类同名方法覆盖了父类的方法。仍然还可以调用父类的方法(super.init)。

  • 错误提示信息：

    指令 作用
    try 为缩进的代码设置陷阱
    except 处理错误的代码
    else 没有出错执行的代码
    finally 无论出错否都执行的代码

  • 算法的五大特性：有穷性，能行性，确定性，输入，输出。

  • 算法具有有穷性，程序不需要具备有穷性。

  • 算法设计的目标：正确性，可读性，健壮性，时间复杂度，空间复杂度。

第四部分 线性表

线性存储

  • 线性结构是n(n≥0)个数据元素构成的有限序列。
  • 所有元素按照逻辑顺序依次存储在一块连续的存储空间中，简称顺序表。
  • Python内置线性结构：list, turple, str。
  • 线性表的顺序存储：存取O(1),称顺序表具有随机存取(Random Access)或直接存取的特性。
  • 优点：
    1. 程序设计简单。
    2. 元素的物理位置反映逻辑关系，可实现随机存取，根据位序的读写时间效率为O(1)。
    3. 存储密度为1。
  • 缺点：
    1. 须事先确定初始表长
    2. 插入删除会带来元素的移动
    3. 多次插入后初始空间耗尽，造成溢出或需要空间扩容

链式存储

  • 单链表，循环链表，双向链表等。(是/否带头结点，单向/双向，是/否循环)
  • 优点：
    1. 动态管理存储空间，不需事先申请空间
    2. 不需要担心溢出
    3. 插入删除只引起指针的变化，效率更高。
  • 缺点：
    1. 顺序存取，非随机存取，根据位序读写效率为O(n)
    2. 由于涉及到指针操作，程序设计的复杂性增大
    3. 链域也占空间，使存储密度降低，必定小于1。

第五部分 递归

To Iterate is Human, to Recurse, Divine.
——L. Peter Deutsc

  • 递归是一种自我嵌套定义方式，可用于对数学概念、实体、问题解决方案等进行定义。

  • Recursion is the process of repeating items in a self-similar way. (wiki)

  • 递归“三定律”：

    1. 必须有递归结束的条件，即一个或多个递归出口，通常对应于最小规模或满足某个条件时无需递归可直接完成的情况，常被称为基本情况(Base Case)。
    2. 原问题可以被分解为规模更小但和原问题有着相同解法的子问题(调用自身)，并逐渐向基本情况演进。
    3. 子问题的个数必须是有限的，相应地，递归调用的次数也必须是有限的，即保证不管多大的规模，都可以最终到达一个基本情况。

  • 实例较多，除了概念没啥其他好说的。
