第四部分 线性表

线性存储

  • 线性结构是n(n≥0)个数据元素构成的有限序列。
  • 所有元素按照逻辑顺序依次存储在一块连续的存储空间中，简称顺序表。
  • Python内置线性结构：list, turple, str。
  • 线性表的顺序存储：存取O(1),称顺序表具有随机存取(Random Access)或直接存取的特性。
  • 优点：
    1. 程序设计简单。
    2. 元素的物理位置反映逻辑关系，可实现随机存取，根据位序的读写时间效率为O(1)。
    3. 存储密度为1。
  • 缺点：
    1. 须事先确定初始表长
    2. 插入删除会带来元素的移动
    3. 多次插入后初始空间耗尽，造成溢出或需要空间扩容

链式存储

  • 单链表，循环链表，双向链表等。(是/否带头结点，单向/双向，是/否循环)
  • 优点：
    1. 动态管理存储空间，不需事先申请空间
    2. 不需要担心溢出
    3. 插入删除只引起指针的变化，效率更高。
  • 缺点：
    1. 顺序存取，非随机存取，根据位序读写效率为O(n)
    2. 由于涉及到指针操作，程序设计的复杂性增大
    3. 链域也占空间，使存储密度降低，必定小于1。

第五部分 递归

To Iterate is Human, to Recurse, Divine. ——L. Peter Deutsc

  • 递归是一种自我嵌套定义方式，可用于对数学概念、实体、问题解决方案等进行定义。
  • Recursion is the process of repeating items in a self-similar way. (wiki)
  • 递归“三定律”：
    1. 必须有递归结束的条件，即一个或多个递归出口，通常对应于最小规模或满足某个条件时无需递归可直接完成的情况，常被称为基本情况(Base Case)。
    2. 原问题可以被分解为规模更小但和原问题有着相同解法的子问题(调用自身)，并逐渐向基本情况演进。
    3. 子问题的个数必须是有限的，相应地，递归调用的次数也必须是有限的，即保证不管多大的规模，都可以最终到达一个基本情况。
  • 实例较多，除了概念没啥其他好说的。

第六部分 查找和排序

查找

线性表查找

  • 查找是指：给定由n个记录(Record)构成的集合(查找表)，其中每个记录都有一个对应的关键字项Key，给定一个目标关键字target，要求在集合中寻找关键字项与target相同的记录。
  • 记录：即数据元素，通常包含若干个数据项。
  • 关键字：也称为键、关键码，是查找时的依据。
  • 在记录构成的集合结构下，以数据检索和查找为主要基本操作，称为查找表、映射、字典或关联。
  • 不同查找表下的查找算法：
  • 平均情况下的关键字比较次数，平均查找长度ASL（Average Search Length），即平均情况下进行一次查找所需进行的关键字比较次数，即对查找表中的每个元素按照某种概率进行查找时的平均比较次数。
  • $ASL = \sum_{i=0}^{n-1}p_{i}c_i$
  • 无序表顺序查找：
    1. $ASL_{成功} = {\frac{1}{n}}\sum_{i=0}^{n-1}{(i + 1)} = {\frac{n + 1}{2}}$
    2. $ASL_{失败} = n$
  • 有序表顺序查找：
    |Case|Best case|Worst Case|Average Case|
    |:—-:|:—-:|:—-:|:—-:|
    |Item is present|1|2n-1|n|
    |Item is not present|2|2n|$n+\frac{2n}{n + 1}$|
  • 有序表二分查找O(logn)。识别相等/不识别相等。

  • 最著名的完美散列函数是MD5和SHA系列函数

    1. 开放定址法(线性探测，二次探测，双哈希函数)

第七部分 树与二叉树

  • 结点的度：每个结点拥有子树的数目被称为结点的度。
  • 叶子结点：度为0的结点被称为叶子结点(也被称为“终端结点”)
  • 分支结点：度不为0的结点被称为分支结点(也被称为“非终端结点”)。
  • 树的度：树内所有结点度的最大值被称为该树的度。

    二叉树

  • 完全二叉树：所含的n个结点和满二叉树中编号为1至n的结点的编号和形状都一一对应。可在满二叉树的基础上连续删除若干个结点得到。
  • 满二叉树：：高度为k且含有$2^k-1$个结点的二叉树。它的叶子结点全部在最下层，分支结点的度全为2。
  • 严格二叉树：也称为2树。是指不存在度为1的结点的二叉树，空二叉树是严格二叉树，非空的严格二叉树中，除了叶子结点之外，每个结点都有2个孩子。
  • 扩充二叉树：对一棵已有的非空二叉树进行扩充，使得原二叉树中所有结点的度数都变为2。空二叉树的扩充二叉树仍然为空。
  • 通过两个序列创建二叉树：
    • 先序和中序(可以)
    • 中序和后序(可以)
    • 中序和层序(可以)
    • 先序和后序(不可以)

拓扑排序

  • 有向无环图(Directed Acyclic Graph)：简称DAG图，是指不存在回路的有向图。
  • 顶点表示活动的网(Activity On Vertex Network)：简称AOV网，是指用顶点表示活动，有向边表示活动发生的先后关系的有向图。可用于表示：工程的施工图、产品生产的流程图、程序的数据流图等，教学课程的依赖图。
  • AOV网一定是DAG图。
  • 拓扑排序：对于一个包含n个顶点的有向图，将所有顶点排成一个线性序列S，要求：如果有向图中存在从顶点u到顶点v的有向边，那么序列S中u必定出现在v之前。

    广度优先搜索拓扑排序：

  • 操作：
    1. 从有向图中选取一个没有前驱的顶点，加入到拓扑序列，接着从图中删去此顶点以及所有以它出发的边。
    2. 重复以上过程直至图中所有顶点加入到拓扑序列中，或者找不到没有前驱的顶点为止。
    3. 这个过程相当于从入度为0的顶点开始，一层层向外扩展的广度优先搜索，因此称为广度优先搜索拓扑排序。
      • bfs拓扑排序判环：如果图中有回路，则在执行到某一步时，再也找不出入度为0的顶点，而拓扑序列中还未包含所有顶点，说明有回路。

深度优先搜索拓扑排序

  • 采用深度优先搜索遍历，若从该顶点开始的遍历已经结束，即它的邻接点已加到序列中了，则将该顶点放在序列的最前面。
  • dfs拓扑排序无法判环。

最小生成树

prim算法

  • 时间复杂度O($n^2$)。
  • 假设G=(V,E)是具有n个顶点的连通网，其最小生成树为T=(U,TE)，其中U是最小生成树的顶点集，TE是最小生成树的边集。
  • 用Prim算法求解最小生成树的过程为：
    1. 初始化生成树，使得U={w}，TE={}，w为图G中任一顶点；
    2. 循环执行n-1次：
      • ①找到一条边(u,v)∈E，这条边(u,v)必须满足2个条件：
        • 条件1：顶点u在生成树中，顶点v不在生成树中(即u∈U，v∈V-U)；
        • 条件2：(u,v)是满足条件1的所有边中权值最小的边；
      • ②将v并入U，将边(u,v)并入TE；

kruskal算法

  • 用到并查集
  • 时间复杂度O(eloge)。
  • 假设G=(V,E)是具有n个顶点的连通网，其最小生成树为T=(U,TE)，其中U是最小生成树的顶点集，TE是最小生成树的边集。
  • 用kruskal算法求解最小生成树的过程为：
    1. 初始化最小生成树，使得U=V，TE={}，即生成树T含有图G的全部顶点，且每个顶点都自成一个连通分量；
    2. 对G的所有边按照权值递增的次序排序；
    3. 依次考察排好序的各条边，若该边的两个顶点属于不同的连通分量，则将该边并入TE，并标记两个顶点所在的连通分量为同一连通分量；
    4. 重复执行步骤3，直到生成树中含有n-1条边，所有顶点在一个连通分量中。

最短路径

单源最短路：Dijkstra算法

  • 用到优先队列。
  • 时间复杂度O(elogn)。
  • 用Dijkstra算法求解单源最短路的过程为：
    1. 初始化distance和pre数组数组，distance[i]为边<0, i>的权值（如果该边不存在，即为∞），即对应于G的邻接矩阵中0行的元素，注意确保distance[0]为0，即表示它已在S集合中。初始化所有的pre[i]为起点0。
    2. 循环执行n-1次：
      • ①从distance数组中找出非0的最小权值所对应的最短路径；
      • ②更新数组中其它非0的distance值和pre值，distance[i]=min(distance[i], distance[k] + 的权值)，如果distance更新为后者，则同时更新pre[i]为k。

多源最短路：Floyd算法

  • 时间复杂度
  • 用Floyd算法求解多源源最短路的过程为
    1. 循环n次，在执行第k(0 ≤ k < n)次循环时：
      依次检查每一个D[i][j],如果i号顶点经过k号顶点到达j号顶点的路径长度更短，则修改D[i][j]为此更小值，同时更新P[i][j]为k号顶点到j号顶点的最短路径的前驱顶点，即更新为P[k][j]。假设经过k号顶点时的路径长度和路径矩阵分别用Dk和Pk表示。
    2. 即：$D^k[i][j]=min(D^{k-1} [i][j], D^{k-1}[i][k]+ D^{k-1}[k][j])$；
    3. 当$D^k[i][j]$更新为$D^{k-1}[i][k]+ D^{k-1}[k][j]$时，则$P^k[i][j]$更新为$P^{k-1}[k][j]$。

关键路径

  • 一个工程的AOE网中，通常只有一个入度为0的顶点，称为源点，对应于工程开始的事件，如开工仪式；通常也只有一个出度为0的顶点，称为汇点，对应于工程结束的事件，如结束典礼。
  • 由于AOE网中某些活动可以并行进行，因此完成整个工程的最短时间即为从源点到汇点最长路径的长度，这条路径称为关键路径，构成关键路径的边即为关键活动。
  • 在工程实践中，可以通过提高关键活动的效率，进而加快整个工程的进程。
